# Contributing

---

## Running Locally

```bash
git clone https://github.com/your-username/realm-v2
cd realm-v2
python scripts/render.py           # Render README from current state
python scripts/engine.py tick      # Run a world tick
python scripts/gh_stats.py your-username  # Test stat fetching
```

No dependencies beyond Python 3.10+ standard library.

---

## Testing a Move

```bash
# Simulate founding a faction
python scripts/engine.py found testuser "Test Civilization"

# Simulate joining
python scripts/engine.py join testuser2 test-civilization

# Simulate an attack
python scripts/engine.py attack testuser 0 1

# Re-render README
python scripts/render.py
```

## Adding a New Biome

1. Add a biome definition to `BIOMES` in `world.py`
2. Add biome-appropriate wonders to `WONDERS` in `world.py`
3. Add the biome key to `biome_seeds` in `state.json`
4. Update `render_biome_legend()` in `render.py` if needed
5. **Update `docs/game-mechanics.md`** with the new biome's terrain table

---

## Adding a New World Event

Add an entry to `WORLD_EVENTS` in `engine.py`:

```python
("🌊", "Description of what happens.")
```

To make it mechanically active (not just narrative), add logic to `world_tick()` or `attack()`.

**Update `docs/game-mechanics.md`** with the new event.

---

## Adding a New Wonder

Add to `WONDERS` in `world.py`:

```python
{"name": "Wonder Name", "emoji": "🏰", "bonus": "bonus_key", "biome": "biome_name"}
```

Add the bonus effect to `apply_wonder_bonus()` in `engine.py`:

```python
elif wb == "bonus_key": bonus["attack"] += X
```

**Update `docs/game-mechanics.md`** with the new wonder.

## Adding a New Action Type

1. Add a new Issue template in `.github/ISSUE_TEMPLATE/`
2. Add a title prefix and parser branch in `process_issue.py`
3. Add the handler function in `engine.py`
4. Add any state fields needed to `state.json`
5. **Update `docs/scripts.md`** and **`docs/state-schema.md`** accordingly

---

## Code Style

- Python 3.10+ standard library only — no external dependencies
- All state mutations go through `engine.py` — `world.py` and `render.py` are read-only relative to game state
- Functions should be pure where possible — take `state` dict, return `(bool, str)` or mutate in-place with clear intent
- Do not hand-edit `README.md` — it is auto-generated by `render.py`

---

## Documentation Rule

**Any change to code must be accompanied by an update to the relevant doc in `docs/`.** This ensures the documentation stays accurate and AI agents working on this codebase have correct context.
